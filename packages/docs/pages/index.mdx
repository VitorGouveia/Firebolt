Firebolt takes the old MVC approach but in a new way, giving you more control over the UI and the Logic. Under the hood we use <a href="https://developer.mozilla.org/en-US/docs/Web/Web_Components/Using_custom_elements">CustomElements</a> to define your components, see a list of <a href="/docs/supported-browsers">supported browsers.</a>

## Overview

```js
const GithubStarButton = firebolt.create("button", () => {
  return {
    component: "github-star-button",
    entry: "main section.github-star-button",
    textContent: "Star the repository",
    onclick(event) {
      this.textContext = "Has already starred";
      this.disabled = true;
    },
  };
});
```

In this example, the `firebolt.create` method accepts a `tag` or a <a href="docs/firebolt-elements">`Firebolt UI Element`</a> and <a href="docs/controllers">`controller`</a> function. `tag` is the element you would like to create or you could create a `Firebolt UI Element` and customize it's UI entirely. the `controller` function is where you define the element `props` and handle `state` and other `things`.

<Callout>
  the `controller` can also just be an object an seen in the example below
</Callout>

```js
const GithubStarButton = firebolt.create("button", {
  component: "github-star-button",
  ...<your props>
});
```

when using `firebolt.create` a new element is `registered` in the DOM, if you just mean to create a UI element and put it in the DOM by hand, you can achieve just that by using `firebolt.createElement`:

```js
const GithubStarButtonUI = firebolt.createElement(() => {
  const { creatorName } = firebolt.useContext(AuthorContext);

  return `
    <button>
      ${creatorName}
    </button>
  `;
});
```

when using `firebolt.createElement` you still can use all the state-related features that firebolt has.

<Callout>
  You can use any HTML renderer when returning from `createElement`, like
  <a href="https://github.com/WebReflection/uhtml">`uHTML`</a> for example
</Callout>

```js
const GithubStarButtonUI = firebolt.createElement(() => {
  return html`hi mom, i'm using uHTML`;
});
```

Then if you want to use the `GithubStarButtonUI` with some logic just pass it as the first parameter like so:

```js
const GithubStarButton = firebolt.create(GithubStarButtonUI, {
  component: "github-star-button",
  ...<your props>
});
```

Firebolt gives you full control over the DOM, so you can use elements the way you want. In this example, the `GithubStarButton` contains `special functions` that you can use to `modify` state, `listen` to updates or change prop values.

## Installing

first, load the script into your HTML or JavaScript file

```html
<script type="module" src="public/firebolt.js">
```

or

```js
import firebolt from "./firebolt.js";
```

then start using firebolt, it's that simple!

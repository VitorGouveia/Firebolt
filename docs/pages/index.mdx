<OnView>
  <h2>Overview</h2>
</OnView>

<OnView>
  ```js
  const GithubStarButton = firebolt.create("button", () => {
    return {
      component: "github-star-button",
      entry: "main section.github-star-button",
      textContent: "Star the repository",
      onclick(event) {
        this.textContext = "Has already starred";
        this.disabled = true;
      },
    };
  });
  ```
</OnView>

<OnView>
  In this example, the `firebolt.create` method accepts a `tag` or a{" "}
  <a href="docs/firebolt-elements">`Firebolt UI Element`</a> and{" "}
  <a href="docs/controllers">`controller`</a> function. `tag` is the element you
  would like to create or you could create a `Firebolt UI Element` and customize
  it's UI entirely. the `controller` function is where you define the element
  `props` and handle `state` and other `things`.
</OnView>

<OnView>
  <Callout>
    the `controller` can also just be an object an seen in the example below
  </Callout>
</OnView>

<OnView>
  ```js
  const GithubStarButton = firebolt.create("button", {
    component: "github-star-button",
    ...<your props>
  });
  ```
</OnView>

<OnView>
  when using `firebolt.create` a new element is `registered` in the DOM, if you
  just mean to create a UI element and put it in the DOM by hand, you can
  achieve just that by using `firebolt.createElement`:
</OnView>

<OnView>
  ```js
  const GithubStarButtonUI = firebolt.createElement(() => {
    const { creatorName } = firebolt.useContext(AuthorContext);
  
    return `
      <button>
        ${creatorName}
      </button>
    `;
  });
  ```
</OnView>

<OnView>
  when using `firebolt.createElement` you still can use all the state-related
  features that firebolt has.
</OnView>

<OnView>
  <Callout>
    You can use any HTML renderer when returning from `createElement`, like
    <a href="https://github.com/WebReflection/uhtml">`uHTML`</a> for example
  </Callout>
</OnView>

<OnView>
  ```js
  const GithubStarButtonUI = firebolt.createElement(() => {
    return html`hi mom, i'm using uHTML`;
  });
  ```
</OnView>

<OnView>
  Then if you want to use the `GithubStarButtonUI` with some logic just pass it
  as the first parameter like so:
</OnView>

<OnView>
  ```js
  const GithubStarButton = firebolt.create(GithubStarButtonUI, {
    component: "github-star-button",
    ...<your props>
  });
  ```
</OnView>

<OnView>
  Firebolt gives you full control over the DOM, so you can use elements the way
  you want. In this example, the `GithubStarButton` contains `special functions`
  that you can use to `modify` state, `listen` to updates or change prop values.
</OnView>

<OnView>
  <h2>Installing</h2>
</OnView>

first, load the script into your HTML or JavaScript file

```html
<script type="module" src="public/firebolt.js">
```

or

```js
import firebolt from "./firebolt.js";
```

then start using firebolt, it's that simple!
